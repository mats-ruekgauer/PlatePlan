Section 3 — Screen Redesigns

What changes on each screen. Global background is already there — these are the per-screen deltas.

Main Tabs

Screen What changes
Meal Plan
(tabs)/index.tsx		 Remove solid bg-white + existing blob Views (replaced by global background). DayCard headers get a subtle glass divider. MealSlot uses finalized GlassCard + new pill badges. "Dein Plan" header stays, week-range text beneath it. Status card at top becomes a GlassCard. FloatingAction becomes a GlassCard pill button.
Shopping
(tabs)/shopping.tsx		 Replace bg-[#F8F9FA] with transparent. Each category group becomes a GlassCard (primary depth). Checked items fade to 0.4 opacity. Progress bar becomes a frosted GlassCard with a green fill track. "All done" banner is a primary GlassCard with gradient accent. Shopping items themselves get a subtle glass row.
Profile
(tabs)/profile.tsx		 Replace all Card usages with GlassCard. Settings sections (Ziele, Präferenzen, Favoriten, Automationen) each become a GlassCard. Switch + TextInput fields get glass styling. Sign out becomes a ghost/danger button. Profile header gets avatar placeholder in a frosted circle.

Auth Screens

Screen What changes
Welcome
(auth)/welcome.tsx		 Global background shows through. App name + tagline centred on the canvas. Two buttons (Anmelden, Registrieren) as GlassCard pill — primary gradient + secondary glass stacked. No other surface elements needed.
Sign-in / Sign-up
(auth)/sign-*.tsx		 Form lives inside a single primary GlassCard centred on screen. TextInputs get a secondary glass style (white/40 bg, white/60 border). Submit = primary gradient button. Back arrow ghost button top-left.

Onboarding (7 steps)

All 7 steps share the same treatment: global background, step content in a primary GlassCard, progress indicator at top as a thin green track on a frosted pill. Continue/Back buttons at bottom use primary gradient + secondary glass. No per-step background changes — consistency is the goal here.

Detail & Settings Screens

Screen What changes
Meal Detail
meal/[id].tsx		 Hero section (recipe title, macros) in primary GlassCard. Ingredients list as secondary GlassCard. Steps as secondary GlassCard. StarRating + feedback form in primary GlassCard at bottom. Back button is a frosted glass circle.
Settings screens
settings/*.tsx		 Same pattern as Profile — all Card → GlassCard, solid backgrounds removed, inputs get glass styling. Consistent with onboarding look since both are edit-form flows.

What is NOT changing

  • All data hooks, React Query keys, Supabase calls — zero business logic changes
  • Component props and interfaces — only internal styling changes
  • Navigation structure — Expo Router file layout stays identical
  • Animations — keeping existing press/active states, no new animation library needed
