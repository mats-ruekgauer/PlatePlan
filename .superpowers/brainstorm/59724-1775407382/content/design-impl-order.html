Section 4 — Implementation Order

Design-system-first approach. Each phase is independently testable on device.

Phase 1 — Foundation
Tokens + Global Background + Tab Bar
  • constants/theme.ts — add glass tokens, bump radius.xl
  • app/_layout.tsx — add persistent gradient + 5 blob Views behind Stack
  • components/ui/GlassCard.tsx — finalise primary/secondary depth prop
  • app/(tabs)/_layout.tsx — replace tab bar with frosted floating pill using Ionicons
Phase 2 — UI Primitives
Button, Badge, Card, Skeleton
  • components/ui/Button.tsx — primary → gradient, secondary → glass tile
  • components/ui/Badge.tsx — unify all variants to pill shape, remove solid bg
  • components/ui/Card.tsx — delegate to GlassCard (thin wrapper)
  • components/ui/Skeleton.tsx — glass-tinted pulse colour
Phase 3 — Plan Components
MealSlot, DayCard, MealSwapper
  • components/plan/MealSlot.tsx — glass cards, pill badges, gradient Bestätigen btn
  • components/plan/DayCard.tsx — German day names, glass day-divider header
  • components/plan/MealSwapper.tsx — frosted glass bottom sheet
Phase 4 — Main Tabs
Meal Plan, Shopping, Profile
  • app/(tabs)/index.tsx — remove old blobs + bg-white, wire up finalised components
  • app/(tabs)/shopping.tsx — category GlassCards, glass progress bar
  • app/(tabs)/profile.tsx — Card → GlassCard throughout, glass inputs
Phase 5 — Auth, Onboarding, Detail
All remaining screens
  • app/(auth)/*.tsx — welcome hero, sign-in/up glass form cards
  • app/(onboarding)/*.tsx — consistent GlassCard step wrapper + progress pill
  • app/meal/[id].tsx — glass hero + ingredient/step cards
  • app/settings/*.tsx — same pattern as Profile
Each phase can be reviewed on device independently. Phases 1–3 de-risk the design system before touching any screen layout.
